小程序开发文档细节-【组件】

1.view 的 hover-class

指定按下去的样式类。当 hover-class=”none” 时，没有点击态效果

首先就在 view 看到的属性名，因为看了很多小程序没有点击效果，没有及时给用户反馈，体验不好，不如统一加一个 class ，我觉得这个是可以有的，前期阶段就确定好，免得之后还要一个个修改，当然也有些组件本身带有这个效果，比如：导航 navigator 就自带这个效果。你可以自定义一个，默认值是

2.scroll-view 的 enable-back-to-top

iOS点击顶部状态栏、安卓双击标题栏时，滚动条返回顶部，只支持竖向

这个方便了，小程序自带，长列表返回也没必要加浮悬球了。

3.text 的 selectable

文本是否可选

这个默认 false 很多小程序内不能复制文本，原因在这里，不过只能选择节点内的文本。

4.button 的 loading

名称前是否带 loading 图标

5.button 的 form-type

用于

组件，点击分别会触发 组件的 submit/reset 事件

6.button 的 open-type=”contact”

直接打开客服对话，当然还有其他的，getphonenumber，launchApp等，便利了很多。

7.checkbox 的 color

checkbox的颜色，同css的color

注意其他的组件也有这个 color 属性，比如：icon，radio，switch 等 ，所以可以满足你各种主题色的修改。

8.form 的 report-submit

是否返回 formId 用于发送模板消息

这里需要特别注意 formId 用于发布模板消息，但是这个模板消息是有时效限制，
模板下发条件：用户本人在微信体系内与页面有交互行为后触发，详见下发条件说明

下发条件说明

支付

当用户在小程序内完成过支付行为，可允许开发者向用户在7天内推送有限条数的模板消息（1次支付可下发3条，多次支付下发条数独立，互相不影响）

提交表单

当用户在小程序内发生过提交表单行为且该表单声明为要发模板消息的，开发者需要向用户提供服务时，可允许开发者向用户在7天内推送有限条数的模板消息（1次提交表单可下发1条，多次提交下发条数独立，相互不影响）

所以怎么收集这个就要前期考虑（比如常见的点击，用 button 伪装一下），以及避免违规。

9.input 的 confirm-type

设置键盘右下角按钮的文字

没错，可以定义键盘右下角的提醒文字了！可以猜想，对于 ios 系统键盘应该是没问题,至于其他第三方，建议都测测看。

10.textarea 的 auto-height 及 show-confirm-bar

是否自动增高，设置auto-height时，style.height不生效

是否显示键盘上方带有”完成“按钮那一栏，1.6.0 支持

11.image 的 宽高默认值

==image组件默认宽度300px、高度225px==

12.video 的 danmu-list

弹幕列表

够用，可以这样说，希望之后增加不知道有没有更好的……但是里面提供的其他属性完全够用了，比如：enable-progress-gesture 开启 video 控制进度的手势。
